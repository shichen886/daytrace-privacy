第三方 SDK 与系统组件清单

适用版本：日迹 0.5.0
更新日期：2026 年 7 月 11 日

本清单根据当前 Flutter 依赖、Android Gradle 依赖、原生代码和合并 Manifest 整理。

Flutter

提供方
Google 与 Flutter 开源社区
使用目的
应用界面、交互和跨平台运行时
处理的数据
在设备内呈现用户输入和本地业务数据
是否联网
日迹正式业务不通过 Flutter 上传用户内容；调试构建会为开发工具通信使用网络
上传用户内容
否
官方说明
Flutter Privacy

sqflite 与 SQLite

提供方
Flutter 社区与 SQLite 项目
使用目的
在应用私有目录保存分类、计划、任务、计时、记录和每日回顾
处理的数据
用户主动创建的本地业务数据及其关联关系
是否联网
否
上传用户内容
否
官方说明
sqflite package，SQLite About

ML Kit 中文文字识别 16.0.1

提供方
Google
使用目的
识别用户主动选择图片中的中文文字
处理的数据
所选图片及本机识别结果；SDK 还可能处理设备信息、应用信息、安装标识、性能指标、API 配置、输入输出大小、事件类型和错误码
是否联网
识别模型随应用打包，识别在设备上完成；SDK 可能联系 Google 获取维护信息并发送加密的运行指标
上传用户内容
输入图片和识别结果不发送给 Google，也不发送给日迹开发者
官方说明
Android 文字识别，条款与隐私，数据披露

Android Storage Access Framework

提供方
Google Android 系统
使用目的
选择 OCR 图片、打开备份文件和选择备份保存位置
处理的数据
用户在系统界面主动选择的图片、JSON 文件或保存目标
是否联网
日迹本身不联网；如果用户选择网盘等文档提供方，其联网行为由该提供方负责
上传用户内容
日迹不上传。文件是否进入第三方存储由用户选择的位置决定
官方说明
Storage Access Framework

未接入的高风险组件

当前版本没有广告 SDK、第三方统计 SDK、崩溃上报服务、社交登录、支付 SDK、云数据库、消息推送或云端 AI SDK。

intl、path 和 cupertino_icons 是格式化、路径处理和图标资源工具，不负责收集或上传用户数据，因此未作为个人数据处理 SDK 单独列项。

当前版本也没有使用 shared_preferences、image_picker、file_picker、path_provider 或 package_info_plus。
